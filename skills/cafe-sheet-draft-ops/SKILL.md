# カフェシート運用

スプレッドシート補完、下書き投入、登録済み判定、行フォーマット更新を一貫した手順で実行する。

## ワークフロー

1. 編集前に対象行を読み込む。
2. 必須項目を先に埋め、その後に任意項目を補完する。
3. 情報源は公式サイト優先、なければ食べログを使う。
4. `アクセス` は食べログの `交通手段` が取れる場合はそれを入れる。
5. `アクセス` に情報源ラベル（例: `（食べログ）`）を追記しない。
6. `主要メニュー` は登録しない（空欄運用）。
7. `アンバサダーコメント` は作業利用の視点で具体的に書く。
8. プロジェクトスクリプトで `cafe_drafts` に投入する。
9. `public.cafes` を照合して登録済みを判定する。
10. 登録済み行をグレーアウトする。

## データルール

- 列仕様は `references/column-rules.md` に従う。
- 時刻フォーマットは `HH:MM` を使う。
- 混雑度の初期値はインポートスクリプトの仕様に合わせる。
- DB照合時の店舗名は、余分な空白と記号を除去して比較する。

## 実行コマンド

```bash
# ドライラン
npm run drafts:import -- --from-row 2 --to-row 120

# 本反映
npm run drafts:import:apply -- --from-row 2 --to-row 120
```

```bash
# 単一行の反映例
npm run drafts:import:apply -- --from-row 37 --to-row 37
```

## 確認項目

1. 対象行が `cafe_drafts` に保存されていること。
2. `public.cafes` 登録済みの行が下書き投入対象から除外されること。
3. 一覧画面の取得上限が運用件数に対して十分であること。
4. グレーアウトが意図した行だけに適用されていること。
